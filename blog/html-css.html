CSS Selectors: .class and #id

The co-dependent relationship between HTML and CSS is evident in the application of type selectors and attribute selectors. Selectors create hooks for CSS to apply cascading style across an HTML document, which is then brought to life on your screen by your chosen web browser. The word cascading means that a style applied to a parent element will also apply to all children elements within the parent (remember html syntax trees?). So with type selection, if you set the colour of the body text to "blue", all headings, paragraphs, and other text elements within the body will also get the same colour (unless you specify something else). You do this by creating attribute selectors, which allow you to assign an “attribute” to an element in your html document to be styled specifically by CSS to meet your design specifications. Although you can achieve some basic styling using simple type selectors (element tags), you will soon run out of formatting options and certainly won’t be able to create a modern looking website. The remainder of this blog will explore best practices when using the attribute selectors .Class and #id (just two of seven!).

.class is a powerful attribute selector which can be used to achieve complex styling because HTML elements can have more than one class assigned to them. Developers should be careful not to create too many novel classes per HTML document as this complexity isn’t considered best practice (careful you don’t create your own document language) and can make it hard for other developers to follow your code. It contains various information about the computer like colour, model, price, discount price while #id is the computer's serial number. It is unique to your product that you take home. What happens if you accidentally create two type selectors of equal importance? If you accidentally create two rules that have equal specificity, the one that comes last in the CSS is the one that will be used.

Who wins the styling battle: type, .class, or #ID?

Type: the most generic selector - always loses. .class: more specific than Type, .class wins over Type. #id: The most specific, beats Type & .class!
